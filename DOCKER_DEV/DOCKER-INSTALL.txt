Aqui está o que os seus testes revelaram:

1. O comando correto para ver a versão
No Rust, o comando para verificar a versão é --version (ou -V maiúsculo). O -v sozinho geralmente espera um arquivo de entrada para compilar.

Tente rodar:

Bash
rustc --version
Ou, para ver informações detalhadas (o que você provavelmente queria com o -v):

Bash
rustc -vV
2. rustc vs rust
Diferente de linguagens como Python ou Julia, o comando principal do compilador é rustc. Não existe um comando rust. Por isso o terminal sugeriu outras ferramentas.

No dia a dia, você raramente usará o rustc diretamente. O padrão da comunidade é usar o Cargo, que é o gerente de pacotes e construtor do Rust. Verifique se ele está aí:

Bash
cargo --version
Resumo do que está acontecendo:
rustc: É o compilador (transforma seu código em um executável). Ele já está no seu PATH, pois respondeu ao primeiro comando.

cargo: É o "canivete suíço". Ele cria projetos, baixa bibliotecas e compila tudo para você.

Dica para o WSL
Como você está no Ubuntu dentro do Windows (WSL), se o rustc --version mostrar uma versão muito antiga (o Ubuntu 24.04 costuma trazer a 1.75), recomendo fortemente usar o rustup para manter tudo atualizado, igual fizemos com o Julia:

Bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
Isso garante que você tenha sempre a versão mais estável (atualmente acima da 1.80) e todas as ferramentas necessárias.
